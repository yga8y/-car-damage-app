# 车辆定损APP项目开发指南

## 项目概述

一款基于AI视觉识别的车辆定损手机APP，用户通过拍照上传车损图片，AI自动分析损伤部位、程度，并基于"只换不修"原则生成配件报价单。

---

## 核心功能流程

```
用户拍照上传 → AI视觉识别（部位+损伤类型+程度） → 只换不修规则判断 → 
配件匹配（车型+OE号） → 价格查询 → 生成报价单（配件费+工时费）
```

---

## 技术架构

### 1. AI视觉识别模块

**开源数据集（可直接使用）：**

| 数据集 | 地址 | 内容 | 格式 |
|--------|------|------|------|
| CarDD | https://github.com/HUANG-Siran/CarDD | 4000+张，6类损伤 | XML/YOLO TXT |
| TQVCD | https://github.com/ULIB-SJTU/TQVCD | 多角度车损 | 标注完整 |
| 安全件专项 | Hugging Face搜索 | 大灯/玻璃/结构件 | 开源 |

**识别目标：**
- 部位：保险杠、车门、大灯、后视镜、叶子板、玻璃等
- 损伤类型：划痕、凹陷、裂纹、破碎、变形
- 程度：轻微/中度/严重

### 2. 只换不修规则引擎

**必须更换的情况：**
1. 安全件损坏（玻璃、大灯、气囊、结构件）→ 坏一点就必须换
2. 裂纹/穿孔 → 必须换
3. 维修费用 > 配件价格50% → 必须换
4. 塑料件/覆盖件严重破损 → 必须换

### 3. 配件数据库结构

**车型库字段：**
```
品牌,车系,年款,排量,变速箱,车身结构,VIN前缀,车型级别
```

**配件库字段：**
```
配件ID,配件分类,配件名称,OE编号,适配车型,安装位置,原厂指导价(元),是否安全件,只换不修规则
```

### 4. 关联逻辑规则（部分示例）

**前保险杠损伤关联：**
- 前保险杠破裂 → 前保险杠骨架、卡扣、内衬、雾灯、雷达探头、下护板、导流板
- 前保险杠严重变形 → 前保险杠骨架、卡扣、内衬、雾灯支架、雷达支架、下护板

**大灯损伤关联：**
- 左前大灯破碎 → 左前大灯支架、前保险杠(左侧)、卡扣(左侧)、水箱上横梁、左前叶子板、左前雾灯、左前雷达探头

## 开发阶段规划

### 第一阶段：MVP（4-6周）

1. **AI模型训练**
   - 下载CarDD数据集
   - 使用YOLOv8训练车损识别模型
   - 目标：识别准确率 ≥90%

2. **基础数据库搭建**
   - 导入文档中的100款车型数据
   - 导入100+常见配件数据
   - 建立基础关联规则（前20条）

3. **APP原型开发**
   - 拍照上传功能
   - 基础AI识别展示
   - 简单报价单生成

### 第二阶段：完善（4-6周）

1. **扩展数据库**
   - 扩充至300+车型
   - 完善配件库（500+配件）
   - 添加工时费标准库

2. **规则引擎完善**
   - 实现全部800+条关联规则
   - 优化只换不修判断逻辑

3. **功能增强**
   - VIN码识别
   - PDF报告导出
   - 历史记录管理

### 第三阶段：商业化（持续）

1. 对接汽配API实时价格
2. 多城市工时费适配
3. 保险理赔对接
4. 维修厂管理系统对接
